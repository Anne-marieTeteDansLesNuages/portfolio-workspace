IDEAL STRUCTURE FOR ANY PROJECT
each project is its own world, thus they each have their dependencies list

myproject/
    .venv/
    manage.py
    project/
    app1/
    app2/
    requirements.txt -> which package are needed, their versions, how to recreate env exactly, everything that s needed for production.
    requirements-dev.txt -> tools used for dev only, like linters, test frameworks, django debug toolbar, anything that s not needed in production
